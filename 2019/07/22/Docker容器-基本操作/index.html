引言

​ docker容器和传统操作系统相比并没有太大的差别，如果是centos系统那是yum，如果是Ubuntu系统那就apt-get，当然如果要制造用于生产的镜像，那么基本上都是基于Alpine系统作为最基础Linux系统进行构建。而在容器内部执行指令之类的操作通常是用于查bug，改配置等。但是在使用的过程中还是有许多地方会和传统的Linux系统在操作上有点不一样，通过熟悉这些最基本的操作，就可以很快的上手docker。

Dockerfile编写

Dockerfile是构建Docker镜像的基础，通过Dockerfile内置的命令构建出我们需要的镜像，当然也可以直接在空白的Linux镜像中像普通的系统一样进行操作，但是并不适合用于生产的程序。Dockerfile基本指令：

FROM：指定基础镜像

RUN：执行命令

COPY：复制文本

ADD：高级复制文件（源路径可以是一个 URL）

CMD：容器启动命令

ENTRYPOINT：入口点（同CMD）

ENV：设置环境变量

ARG：构建参数

VOLUME：定义匿名卷

EXPOSE：暴露端口

USER：指定当前用户（通常生产环境要指定非root用户）

这是一段基本的构建java程序docker镜像的dockerfile命令，当然里面有一些自己的的环境变量之类的。
